1. Introduction

1.1 Purpose

The purpose of the Book Reviewer web application is to provide users with a platform where they can efficiently manage their reading activities, organize book collections, write and share reviews, and engage in discussions about literature. The app aims to enhance the reading experience by enabling users to discover new books, connect with other readers, and express their opinions on literary works.

1.2 Suggestions for browsing the document

For a better understanding of this document, it is recommended to go through it in order subunits presented in the inside-content above.

1.3 Product Purpose

The purpose of the Book Reviewer platform is to empower users to engage with literature by creating detailed reviews and discussions on various books. Through this platform, users can share their thoughts and insights, fostering a community where readers can exchange opinions and recommendations. The Book Reviewer aims to enhance users' understanding and appreciation of books by providing a platform that encourages thoughtful analysis and exploration of literary works.

2. General Description

2.1 Product's Perspective

This application consists of three components: a main web page that can be viewed by any user, a set of web pages available only to users with a created account, and a database to store all user-related data.

2.2 Product's Functions

The web application is expected to perform the following functionalities, depending on the users:

  • Landing page with general description
  • Authentication or registration of the users
  • Viewing and editing profiles
  • Creating book reviews
  • Completing reviews
  • Centralizing review data and generating statistics
  • Administration page of the platform

2.3 Classes and Characteristics of Users

The two categories of users interacting with the application are differentiated by the services they have access to.

The first category represents visitors. They have access to the landing page, authentication, and registration functionalities. Additionally, they can view the reviews list.

The second category consists of authenticated users, who are divided into two subcategories:

  • Users: They can create/complete reviews and have access to resulting statistics.
  • Admins: They manage users/roles, handle problematic reviews, and can send platform-related announcements.

2.4 Operating Environment

The application utilizes web technologies without using frameworks. HTML, CSS, and JavaScript handle front-end development, while Node.js powers the back-end.

3. External Interface Requirements

3.1 User Interfaces

Any visitor can access a limited range of services, including the login/registration section, as well as adding books to your bookshelfs and creating book reviews. For registration, the following information will be requested: name, email address, password and confirmation, while for authentication, only the email address and password are required.

After authentication, the user will be directed to a specific page based on their role.

  • User -> will see the book reviews feed page and will have access to the following pages: my books, profile, create reviews, edit review.
  • Admin -> will see the admin dashboard page and will also have access to the profile page, review statistics and book review's feed.

3.2 Hardware Interfaces

The application interface has been designed to be compatible with the most commonly used devices such as mobile phones, tablets, and PCs. Given the application's functionalities, a low cost of technical requirements and a reduced backend capacity are estimated.

3.3 Communications Interfaces

The communication between the front-end and back-end relies on the HTTP protocol. Requests to the database will be made using SQL queries.

4. System Features

4.1 Registration

Description

In order to create, complete, and review books, users of the application must first create an account. The required information for creating an account will be a name, a unique valid email, and a password that meets certain security conditions.

Stimulus/Response Sequences

The steps a user must follow to create an account are as follows:

  • Enter the name in the corresponding field.
  • Enter a unique valid email address in the corresponding field.
  • Enter a password that meets certain requirements.
    The user will be informed in real-time which of these requirements are met by the text entered in the password field.
  • Re-enter the password (exactly the same) from the previous field.
  • Click the "Sign Up" button below the input fields.

After clicking the "Sign up" button, if any of the fields have not been completed (correctly), the account will not be created, and the user will be alerted to that specific field. If the registration is successful, the user will receive a message informing them that an email has been sent to them for identity confirmation.

Functional Requirements

To create the account, each field must be completed, and the following fields must also have valid content:

  • The email must be valid and unique.
  • The password must meet the following requirements: contain at least 8 characters.
  • The text entered in the password confirmation field must match the text in the corresponding password field.

4.2 Authentication

Description

The role of this component is similar to the registration component. However, it is not enough for a user to have an account in the application, they must also authenticate with this account to benefit from the application's functionalities.

Stimulus/Response Sequences

The steps a user must follow to authenticate are as follows:

  • Enter the email address used when creating the account.
  • Enter the password set when creating the account.
  • Click the "Log In" button.

Functional Requirements

To authenticate, the combination of email and password must be found in the application's database.

4.3 Book Management

Search and Browse Books

  • Explore books by categories, genres, authors, publishers, and publication years.
  • Utilize advanced search filters to find specific books based on criteria like language or series.

Personal Library Management

  • Add new books to personal libraries.
  • Edit book details (e.g., title, author, genre) within the library.
  • Remove books from the library when no longer desired.

4.4 Reading Progress

Book Status Tracking

  • Mark books as "currently reading," "read," or "want to read" to organize reading lists.
  • Update and track progress within each reading status (e.g., percentage completed, last read page).

Reading History

  • View a comprehensive history of books read, including completion dates and reading durations.

4.5 News

Book-Related Announcements

  • Subscribe to RSS feeds for book-related announcements, such as new releases, author interviews, and literary events.
  • Receive notifications about updates to favorite authors or series.

4.6 Statistics

Reading Habits Analysis

  • Generate statistics on reading habits, such as:
  • Most read genres and authors.
  • Average time spent reading per session or per book.
  • Reading progress over time (e.g., monthly or yearly summaries).

Data Export

  • Export reading data and statistics in CSV and DocBook formats for further analysis or archival purposes.
